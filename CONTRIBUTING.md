# Contributing

This repo is one operator's field notes and a reproducible build path — not a software project in the traditional sense. Contributions that fit:

- **Corrections** — a wrong claim, a broken link, a typo. Open an issue or a PR directly.
- **Additional field notes** — you hit a platform-specific gap (macOS/Linux equivalents of [Windows Reality](docs/04-windows-reality.md)?). Open an issue with what broke and what fixed it.
- **"In the Wild" entries** — built something because you followed this thesis? Open an issue with what it does, what gap it closes, and a link. See [docs/06-in-the-wild.md](docs/06-in-the-wild.md) for the format and tone.
- **Translations** — this repo is EN/DE today. A third language is welcome if you're willing to maintain it (translations rot; an abandoned half-translated language is worse than none).

What doesn't fit: swapping in a different "pillar 1/2/3" project as *the* canonical reference. The thesis is tool-agnostic (see the [FAQ](docs/05-faq.md)), but this repo's own build path documents the three projects the author actually uses and can vouch for.

## Style

Match the existing voice: concrete over hype, honest about gaps, field notes rather than marketing copy. If a claim can't be backed by something reproducible, say so plainly instead of smoothing it over.

## Language

Write issues and PRs in English or German — both are read here.

## Deutsch

Dieses Repo ist die Feldnotizen eines Operators und ein reproduzierbarer Bau-Pfad — kein Software-Projekt im klassischen Sinn. Passende Beiträge:

- **Korrekturen** — eine falsche Behauptung, ein kaputter Link, ein Tippfehler. Issue oder PR direkt.
- **Zusätzliche Feldnotizen** — du bist auf eine plattformspezifische Lücke gestoßen (macOS-/Linux-Äquivalente zu [Windows-Realität](docs/de/04-windows-reality.md))? Issue öffnen mit was kaputt ging und was es fixte.
- **„In der Praxis"-Einträge** — etwas gebaut, weil du dieser These gefolgt bist? Issue öffnen mit was es macht, welche Lücke es schließt, und einem Link. Siehe [docs/de/06-in-the-wild.md](docs/de/06-in-the-wild.md) für Format und Ton.
- **Übersetzungen** — dieses Repo ist heute EN/DE. Eine dritte Sprache ist willkommen, wenn du bereit bist, sie zu pflegen (Übersetzungen verrotten; eine verlassene halb-übersetzte Sprache ist schlimmer als keine).

Was nicht passt: ein anderes „Säule 1/2/3"-Projekt als *das* kanonische Referenzprojekt einzutauschen. Die These ist werkzeugagnostisch (siehe [FAQ](docs/05-faq.md)), aber der eigene Bau-Pfad dieses Repos dokumentiert die drei Projekte, die der Autor tatsächlich nutzt und für die er bürgen kann.

### Stil

Triff die bestehende Stimme: konkret statt Hype, ehrlich über Lücken, Feldnotizen statt Marketing-Text. Wenn eine Behauptung nicht durch etwas Reproduzierbares gedeckt ist, sag das offen, statt es zu glätten.

### Sprache

Schreib Issues und PRs auf Englisch oder Deutsch — beides wird hier gelesen.
